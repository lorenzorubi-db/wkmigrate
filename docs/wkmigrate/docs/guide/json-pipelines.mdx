# Working with Exported Pipelines

If you have ADF pipeline definitions as JSON files — exported from the ADF portal, extracted from an ARM template, or checked into source control — you can translate them without Azure credentials using `JsonFactoryDefinitionStore`.

<Admonition type="info" title="When to use this">
Use `JsonFactoryDefinitionStore` when you already have pipeline JSON files locally and do not need to connect to Azure Data Factory at runtime. For live connections to ADF, use `FactoryDefinitionStore` as described in [End-to-End Pipeline Migration](./end-to-end-migration).
</Admonition>

## Directory layout

Place your JSON files in a single directory. The store identifies file types by naming convention:

| Pattern | Resolved as |
|---|---|
| `*trigger*.json` | Trigger definitions |
| `*dataset*.json` | Dataset definitions |
| `*linked_service*.json` | Linked-service definitions |
| Any other `*.json` | Pipeline definitions |

Only pipeline files are required. Missing triggers, datasets, or linked services are skipped with a warning — the pipeline will still translate, but activities that depend on missing resources will produce placeholder tasks.

## Load and translate a pipeline

```python
from wkmigrate.definition_stores.json_factory_definition_store import JsonFactoryDefinitionStore

store = JsonFactoryDefinitionStore(
    definition_dir="/path/to/pipeline/jsons",
)

# List available pipelines
pipeline_names = store.list_pipelines()

# Load and translate a single pipeline
pipeline_ir = store.load(pipeline_names[0])
```

The returned `pipeline_ir` is the same internal representation produced by `FactoryDefinitionStore.load()` and can be passed to any `WorkspaceDefinitionStore` method.

## Batch translation

```python
# Translate all pipelines (failures are logged and skipped)
all_pipelines = store.load_all()
```

Once translated, the `pipeline_ir` object can be passed to any `WorkspaceDefinitionStore` method — creating asset bundles, deploying jobs, or writing local artifacts — as described in [End-to-End Pipeline Migration](./end-to-end-migration). See [`examples/example_json_pipeline.py`](https://github.com/ghanse/wkmigrate/blob/main/examples/example_json_pipeline.py) for a complete working example.
