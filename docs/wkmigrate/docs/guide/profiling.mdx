# Profiling a Data Factory

You can profile an Azure Data Factory resource to assess wkmigrate's support. The profiler inspects all pipelines, datasets, linked services, triggers, and integration runtimes in the factory and classifies each object as *supported* or *unsupported* 
based on wkmigrate's translation capabilities.

<Admonition type="info" title="When to profile">
Run the profiler before starting a migration to understand scope, identify unsupported constructs early, and plan remediation for any objects that require manual migration.
</Admonition>

## Create a Factory Client

The profiler uses a `FactoryClient` to connect to Azure Data Factory. Authenticate with an Azure service principal that has read access to the factory.

```python
from wkmigrate.clients.factory_client import FactoryClient

factory_client = FactoryClient(
    tenant_id="<YOUR AZURE TENANT ID>",
    client_id="<YOUR SERVICE PRINCIPAL CLIENT ID>",
    client_secret="<YOUR SERVICE PRINCIPAL CLIENT SECRET>",
    subscription_id="<YOUR AZURE SUBSCRIPTION ID>",
    resource_group_name="<RESOURCE GROUP WITH YOUR ADF INSTANCE>",
    factory_name="<YOUR ADF FACTORY NAME>",
)
```

<Admonition type="warning" title="Permissions">
Your service principal must have permissions to list pipelines, datasets, linked services, triggers, and integration runtimes. The **Data Factory Reader** role provides adequate permissions for profiling.
</Admonition>

## Run the profiler

Pass the authenticated client to `profile_factory` to generate a `FactoryProfile` containing counts and details for every object in the factory.

```python
from wkmigrate.profiler import profile_factory

profile = profile_factory(factory_client)
```

## View profiler results

Use `format_profile` to render a human-readable summary.

```python
from wkmigrate.profiler import format_profile

print(format_profile(profile))
```

## Work with the profile programmatically

The `FactoryProfile` object exposes structured fields that you can inspect in code.

```python
# Overall counts
print(f"Total activities: {profile.activities.total}")
print(f"Supported: {profile.activities.supported}")
print(f"Unsupported: {profile.activities.unsupported}")

# List unsupported types that need manual migration
for activity_type in profile.unsupported_activity_types:
    print(f"  Needs manual migration: {activity_type}")

# Inspect dataset-to-linked-service mapping
for detail in profile.dataset_details:
    print(f"  {detail.dataset_name} -> {detail.linked_service_name} ({detail.linked_service_type})")
```

## Supported types

The profiler classifies objects using the same type registries as the translator. See [Capabilities](/docs/capabilities) for the full list of supported activities, datasets, and linked services.
